# Spark Connect Scala 3 REPL Guide

Spark Connect Scala 3 provides an [Ammonite](https://ammonite.io/)-based interactive REPL that connects directly to a Spark Connect server for data exploration and development.

## Prerequisites

| Component | Requirement |
|-----------|-------------|
| JDK | 17+ |
| SBT | 1.10+ |
| Scala | 3.3.7 LTS (managed automatically by SBT) |
| Spark Connect Server | 4.0.2+, started and listening |

Start the Spark Connect Server:

```bash
$SPARK_HOME/sbin/start-connect-server.sh
```

## Launching the REPL

### Option 1: Default connection (localhost:15002)

```bash
build/sbt run
```

### Option 2: Specify the full URL via `--remote`

```bash
build/sbt "run --remote sc://myhost:15002"
```

### Option 3: Specify `--host` / `--port` separately

```bash
build/sbt "run --host myhost --port 15002"
```

### Option 4: Use environment variables

```bash
SPARK_REMOTE=sc://myhost:15002 build/sbt run
```

### Connection priority

When multiple options are provided, they are resolved in the following order:

1. `--remote` CLI argument (highest)
2. `SPARK_REMOTE` environment variable
3. URL constructed from `--host` / `--port` etc. (defaults to `sc://localhost:15002`)

## CLI argument reference

| Argument | Value | Description |
|----------|-------|-------------|
| `--remote REMOTE` | `sc://host:port` | Full URI of the Spark Connect Server |
| `--host HOST` | hostname | Server address, defaults to `localhost` |
| `--port PORT` | port number | Server port, defaults to `15002` |
| `--use_ssl` | (no value) | Enable SSL connection |
| `--token TOKEN` | string | Authentication token |
| `--user_id USER_ID` | string | Connection user ID |
| `--user_name USER_NAME` | string | Connection user name |
| `--session_id SESSION_ID` | string | Specify a session ID (auto-generated by default) |
| `--option KEY=VALUE` | key/value | Pass a Spark configuration entry; may be repeated |

Example — with authentication and configuration entries:

```bash
build/sbt "run --host spark.example.com --port 15002 --use_ssl --token my-secret --option spark.sql.shuffle.partitions=10"
```

## What is preloaded in the REPL

After startup, the REPL automatically executes the following predef code:

```scala
import org.apache.spark.sql.functions.*    // 542 built-in functions
import org.apache.spark.sql.implicits.*    // $"col", Seq[T].toDS/toDF, etc.
given SparkSession = spark                  // Implicit SparkSession (used by toDS/toDF)
spark.registerClassFinder(...)             // Auto-upload classes defined in the REPL to the server
```

Objects available out of the box:

| Name | Type | Description |
|------|------|-------------|
| `spark` | `SparkSession` | The connected Spark session |
| `repl` | Ammonite REPL API | Ammonite's internal REPL object |

## Usage examples

### SQL queries

```scala
scala> spark.sql("SELECT 1 + 1 AS result").show()
+------+
|result|
+------+
|     2|
+------+

scala> spark.sql("SHOW DATABASES").show()
```

### DataFrame operations

```scala
scala> val df = spark.range(10)
scala> df.select(col("id"), (col("id") * 2).as("doubled")).show()
+---+-------+
| id|doubled|
+---+-------+
|  0|      0|
|  1|      2|
|  2|      4|
...

scala> df.filter(col("id") > 5).orderBy(col("id").desc).show()
```

### Using `$"col"` syntax

```scala
scala> val df = spark.range(10)
scala> df.filter($"id" > 5).select($"id", ($"id" % 3).as("mod3")).show()
```

### Reading data

```scala
scala> val df = spark.read.format("parquet").load("/path/to/data.parquet")
scala> df.printSchema()
scala> df.describe().show()
```

### Writing data

```scala
scala> spark.range(100)
     |   .withColumn("name", lit("test"))
     |   .write
     |   .mode("overwrite")
     |   .parquet("/tmp/output")
```

### Creating a temp view and querying it

```scala
scala> spark.range(10).createOrReplaceTempView("numbers")
scala> spark.sql("SELECT id, id * id AS squared FROM numbers WHERE id > 3").show()
```

### Dataset[T] — type-safe operations

```scala
scala> case class Person(name: String, age: Int) derives Encoder

scala> val people = Seq(Person("Alice", 30), Person("Bob", 25)).toDS
scala> people.filter(_.age > 26).show()
+-----+---+
| name|age|
+-----+---+
|Alice| 30|
+-----+---+
```

> Note: `derives Encoder` is Scala 3 compile-time derivation, so `import spark.implicits._` is not needed to obtain an Encoder (but the `toDS` method still requires `given SparkSession`, which is already provided by the predef).

### Aggregation

```scala
scala> val df = spark.range(100)
scala> df.groupBy(($"id" % 10).as("group"))
     |   .agg(count("*").as("cnt"), sum("id").as("total"))
     |   .orderBy($"group")
     |   .show()
```

### UDFs (user-defined functions)

Classes and functions defined in the REPL are automatically uploaded to the server through `AmmoniteClassFinder`:

```scala
scala> val double = udf((x: Int) => x * 2)
scala> spark.range(5).select(col("id"), double(col("id")).as("doubled")).show()
+---+-------+
| id|doubled|
+---+-------+
|  0|      0|
|  1|      2|
|  2|      4|
|  3|      6|
|  4|      8|
+---+-------+
```

Register a globally available SQL function:

```scala
scala> spark.udf.register("my_double", (x: Int) => x * 2)
scala> spark.sql("SELECT id, my_double(id) AS doubled FROM range(5)").show()
```

### Window functions

```scala
scala> import org.apache.spark.sql.expressions.Window

scala> val w = Window.orderBy($"id")
scala> spark.range(5)
     |   .withColumn("running_sum", sum($"id").over(w))
     |   .show()
```

### Structured Streaming (interactive exploration)

```scala
scala> val stream = spark.readStream
     |   .format("rate")
     |   .option("rowsPerSecond", "5")
     |   .load()

scala> val query = stream.writeStream
     |   .format("console")
     |   .outputMode("append")
     |   .start()

scala> query.isActive   // true
scala> query.stop()
```

### Catalog operations

```scala
scala> spark.catalog.listDatabases().show()
scala> spark.catalog.listTables().show()
scala> spark.catalog.tableExists("my_table")
```

### Runtime configuration

```scala
scala> spark.conf.get("spark.sql.shuffle.partitions")
scala> spark.conf.set("spark.sql.shuffle.partitions", "100")
```

## Differences from the upstream Spark REPL

| Feature | Upstream Spark REPL | SC3 REPL |
|---------|---------------------|----------|
| Scala version | 2.13 | 3.3.7 |
| Wildcard import | `import functions._` | `import functions.*` |
| Implicit session | `import spark.implicits._` | `given SparkSession = spark` |
| Encoder derivation | Runtime reflection | Compile-time `derives Encoder` |
| `ExtendedCodeClassWrapper` | Yes (registers OuterScopes) | No (uses `CodeClassWrapper`) |
| Embedded Spark server | Supports `withLocalConnectServer` | Not supported (pure client) |
| `spark-shell` script | Yes | No (launched via `build/sbt run`) |

## FAQ

### Q: Connection times out / cannot connect

Make sure the Spark Connect Server is running and the port is reachable:

```bash
# Check whether the server is listening
nc -zv localhost 15002

# Start the server
$SPARK_HOME/sbin/start-connect-server.sh
```

### Q: A case class defined in the REPL throws ClassNotFoundException inside a UDF

SC3's `AmmoniteClassFinder` automatically uploads classes compiled in the REPL to the server. If the error persists:

1. Verify the class definition was executed successfully in the REPL (no compile errors)
2. Verify `spark.registerClassFinder(...)` was executed in the predef (no errors at startup)
3. Every type referenced by the UDF must be defined in the REPL or available on the classpath

### Q: How do I exit the REPL?

Type `exit` or press `Ctrl+D`.

### Q: How do I add an external JAR?

```scala
scala> spark.addArtifact("/path/to/my-lib.jar")
```

### Q: How do I connect over SSL?

```bash
build/sbt "run --host secure-host --port 15002 --use_ssl --token my-token"
```

Or via the `--remote` URL parameter:

```bash
build/sbt "run --remote sc://secure-host:15002;use_ssl=true;token=my-token"
```
